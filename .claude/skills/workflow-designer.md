# Skill: Workflow Designer

> Dify 工作流设计专家

## 触发条件

当用户要求：
- 设计新的工作流
- 修改现有工作流
- 调试工作流问题
- 优化工作流性能

## 工作流程

### 1. 需求分析
- 明确工作流的输入和输出
- 识别关键处理步骤
- 确定需要的知识库

### 2. 节点设计
按照 Dify 工作流规范设计节点：

```
开始节点 → [处理节点]* → [条件分支]? → 结束节点
```

常用节点类型：
- **HTTP 请求**: 获取外部数据
- **知识库检索**: RAG 查询
- **LLM**: 内容生成/处理
- **条件分支**: 逻辑判断
- **变量聚合**: 数据整合

### 3. Prompt 设计
参考 `src/prompts/` 中的模板

### 4. 测试验证
- 准备测试用例
- 验证输出质量
- 记录问题和改进

## 输出规范

工作流设计文档应包含：
1. 流程图
2. 节点配置详情
3. Prompt 模板
4. 测试用例
5. 已知限制
