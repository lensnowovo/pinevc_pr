# PineVC-PR 系统架构 v1.5

> 架构师维护 | 最后更新: 2026-03-01
> **架构决策**: Coze 为主，OpenClaw 补充

---

## 1. 系统概述

### 1.1 项目目标

构建「松禾医健全媒体品牌中央厨房」，实现：
- 从「追热点」升级为「造观点」
- 从「PR 执行」升级为「品牌战略指挥」
- 1 人团队 = 20 人全媒体公关效果

### 1.2 架构决策记录

**ADR-001: 技术平台选型 (v1.5)**

| 方案 | 定位 | 适用场景 |
|------|------|----------|
| **Coze (扣子)** | 主力生产引擎 | 标准化内容生产、工作流自动化 |
| **OpenClaw (龙虾)** | 补充执行引擎 | 最后五厘米：发布、跨软件操作 |

**决策理由**:
1. 核心需求是"稳定产出"，Coze 更可靠
2. Coze 成本极低，适合高频迭代
3. OpenClaw 适合"操作电脑"，但不适合稳定的生产流程
4. 两者结合：Coze 生产 → OpenClaw 执行发布

### 2.2 核心工作流

#### WF-B: 老板转发 → 内容加工

```
触发: 用户在 Coze 发送文章链接
平台: Coze 智能体

流程:
┌─────────────────────────────────────────────────────────────┐
│  1. 接收链接                                                 │
│     └→ 用户发送微信文章链接                                  │
│                                                             │
│  2. 文章解析                                                 │
│     └→ 提取标题、正文、作者、来源                            │
│                                                             │
│  3. 论点匹配                                                 │
│     └→ 检索知识库，匹配 CDE+AI 论点                          │
│                                                             │
│  4. 企业关联                                                 │
│     └→ 检索被投企业库，匹配相关企业                          │
│                                                             │
│  5. 内容生成                                                 │
│     └→ 按品牌模板生成公众号文章                              │
│                                                             │
│  6. 输出                                                     │
│     └→ 返回 Markdown 格式文章                                │
│     └→ 用户复制到公众号后台                                  │
└─────────────────────────────────────────────────────────────┘

输入: { article_url: "..." }
输出: { title: "...", content: "Markdown..." }
```

#### WF-A: 热点监控 → 内容生成

```
触发: 定时 (每日凌晨)
平台: Coze 工作流

流程:
┌─────────────────────────────────────────────────────────────┐
│  1. RSS 抓取                                                 │
│     └→ 从预设信息源抓取最新内容                              │
│                                                             │
│  2. 筛选过滤                                                 │
│     └→ 按相关性、时效性评分                                  │
│                                                             │
│  3. 生成内容                                                 │
│     └→ 对高价值热点生成 PR 内容                              │
│                                                             │
│  4. 推送审核                                                 │
│     └→ 发送给用户确认                                        │
└─────────────────────────────────────────────────────────────┘
```

```

┌─────────────────────────────────────────────────────────────┐
│  1. 汇总隔夜新闻                                             │
│     └→ 从热点池提取                                          │
│                                                             │
│  2. 生成早报                                                 │
│     └→ 按早报模板生成                                        │
│                                                             │
│  3. 推送                                                     │
│     └→ 发送到社群                                            │
└─────────────────────────────────────────────────────────────┘
```

## 3. Coze 配置详解

### 3.1 智能体设计

```
智能体: 松禾 PR 助手

角色设定:
你是松禾资本医健团队的 PR 内容专家。

你的职责:
1. 将输入的文章转化为高质量的公众号内容
2. 确保内容关联年度核心论点 (CDE+AI)
3. 引用被投企业案例作为佐证
4. 保持松禾的品牌形象

工作流程:
1. 解析文章内容
2. 匹配论点框架
3. 关联被投企业
4. 生成符合规范的内容

约束条件:
• 不使用夸张词汇
• 不做股价预测
• 确保信息准确
```

### 3.2 知识库配置

```
知识库 1: 年度论点框架
├── C (Clinical) - 临床转化
├── D (Discovery) - 创新来源
├── E (Engineering) - 工程验证
└── AI - 智能加速

知识库 2: 被投企业信息
├── 企业基本信息
├── 论点标签
├── 赛道标签
└── 近期动态

知识库 3: 品牌规范
├── VI 规范
├── 语调指南
└── 禁用词列表
```

### 3.3 技能/插件

```
必要插件:
├── 链接读取 - 解析文章内容
├── 知识库检索 - 查询论点和企业
└── 内容生成 - 生成 PR 文章

可选插件:
├── RSS 订阅 - 热点监控
└── 定时触发 - 早报生成
```

## 4. OpenClaw 补充场景

### 4.1 适用场景

```
OpenClaw 负责"最后五厘米":

1. 公众号发布
   • 登录公众号后台
   • 创建图文消息
   • 粘贴内容
   • 点击发布

2. 需要登录的网页抓取
   • 登录后的研报平台
   • 需要验证的媒体后台

3. 跨软件操作
   • Excel → 微信
   • 浏览器 → 飞书
```

### 4.2 与 Coze 的协作

```
协作模式:

Coze 生成内容
     │
     ▼
存入中间存储 (飞书文档/Notion)
     │
     ▼
OpenClaw 读取并发布

示例流程:
1. Coze 生成文章 → 保存到飞书文档
2. OpenClaw 定时检查飞书文档
3. 发现新内容 → 登录公众号发布
```

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 内容质量不稳定 | 品牌形象 | 人工终审 + Prompt 优化 |
| 知识库信息过时 | 引用错误 | 定期更新机制 |
| 微信接口限制 | 发布受阻 | 手动发布 + OpenClaw 补充 |
| AI 幻觉 | 信息错误 | 事实核查 + 来源标注 |
