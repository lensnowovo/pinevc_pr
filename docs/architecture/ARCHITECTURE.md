# PineVC-PR 系统架构 v1.0

> 架构师维护 | 最后更新: 2026-03-02
> **架构决策**: Dify (内容生成) + n8n (调度集成)

---

## 1. 系统概述

### 1.1 项目目标

构建「松禾医健全媒体品牌中央厨房」，实现：
- 从「追热点」升级为「造观点」
- 从「PR 执行」升级为「品牌战略指挥」
- 1 人团队 = 20 人全媒体公关效果

### 1.2 架构决策记录

**ADR-001: 技术平台选型**

| 平台 | 定位 | 职责 |
|------|------|------|
| **Dify** | 内容生成引擎 | LLM 工作流、知识库 RAG、内容生成 |
| **n8n** | 调度集成引擎 | 定时任务、数据采集、外部 API 集成 |

**选型理由**:
1. **Dify**: AI 原生设计，内置 RAG 和工作流，适合内容生成
2. **n8n**: 开源自动化平台，200+ 连接器，适合调度和集成
3. **两者结合**: Dify 专注内容质量，n8n 专注流程自动化
4. **完全开源**: 可自托管，数据安全可控

## 3. 核心工作流

### 3.1 WF-B: 老板转发 → 内容加工

```
触发: 手动 (用户输入链接)
主引擎: Dify
辅助: n8n (可选，用于获取微信文章)

Dify 工作流:
┌─────────────────────────────────────────────────────────────┐
│  [开始]                                                      │
│     │                                                        │
│     ▼                                                        │
│  [HTTP请求] 获取文章内容                                     │
│     │ (或使用 n8n 代理获取微信文章)                          │
│     ▼                                                        │
│  [知识库检索] 匹配论点框架                                   │
│     │ Query: 文章内容                                        │
│     │ KB: thesis-framework                                   │
│     ▼                                                        │
│  [知识库检索] 匹配被投企业                                   │
│     │ Query: 文章内容 + 匹配的论点                           │
│     │ KB: portfolio                                          │
│     ▼                                                        │
│  [LLM节点] 生成内容                                          │
│     │ Model: Claude                                          │
│     │ Prompt: 文章加工模板                                   │
│     ▼                                                        │
│  [LLM节点] 质量检查                                          │
│     │ 检查: 论点关联、企业引用、品牌规范                     │
│     ▼                                                        │
│  [结束] 输出 Markdown 文章                                   │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 WF-A: 热点监控 → 内容生成

```
触发: 定时 (每日凌晨 1:00)
主调度: n8n
内容引擎: Dify

n8n 工作流:
┌─────────────────────────────────────────────────────────────┐
│  [Cron触发] 每日凌晨 1:00                                   │
│     │                                                        │
│     ▼                                                        │
│  [HTTP请求] RSS 抓取行业媒体                                │
│     │                                                        │
│     ▼                                                        │
│  [代码节点] 解析 RSS，提取内容                              │
│     │                                                        │
│     ▼                                                        │
│  [HTTP请求] 调用 Dify API                                   │
│     │ POST /workflows/run                                    │
│     │ Body: { events: [...] }                                │
│     ▼                                                        │
│  [条件分支] 检查生成结果                                    │
│     ├── 成功 → [HTTP请求] 推送到审核队列                    │
│     └── 失败 → [日志] 记录错误                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 WF-E: 每日早报

```
触发: 定时 (每日 8:00)
主调度: n8n
内容引擎: Dify

n8n 工作流:
┌─────────────────────────────────────────────────────────────┐
│  [Cron触发] 每日 8:00                                       │
│     │                                                        │
│     ▼                                                        │
│  [读取] 从数据库/文件读取隔夜新闻                           │
│     │                                                        │
│     ▼                                                        │
│  [HTTP请求] 调用 Dify 早报生成工作流                        │
│     │                                                        │
│     ▼                                                        │
│  [条件分支] 检查生成结果                                    │
│     ├── 成功 → [HTTP请求] 推送到社群                        │
│     └── 失败 → [通知] 告警                                  │
└─────────────────────────────────────────────────────────────┘
```

## 8. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Dify 部署复杂 | 延迟启动 | 准备 Docker Compose 一键部署 |
| n8n 学习曲线 | 延迟开发 | 先用简单工作流，逐步复杂化 |
| 内容质量不稳定 | 品牌形象 | 人工终审 + Prompt 优化 |
| 知识库信息过时 | 引用错误 | 定期更新机制 |

## 9. 下一步行动

### 立即执行

1. **部署 Dify**
   ```bash
   git clone https://github.com/langgenius/dify.git
   cd dify/docker
   docker-compose up -d
   ```

2. **部署 n8n**
   ```bash
   docker run -it --rm --name n8n -p 5678:5678 -v ~/.n8n:/home/node/.n8n n8nio/n8n
   ```

3. **准备知识库数据**
   - 整理论点框架
   - 整理被投企业信息
