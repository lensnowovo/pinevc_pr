# PineVC-PR 系统架构 v2.0

> 架构师维护 | 最后更新: 2026-03-01
> **架构决策**: 从 Dify 转向 OpenClaw (龙虾/元气AI)

---

## 1. 系统概述

### 1.1 项目目标

构建「松禾医健全媒体品牌中央厨房」，实现：
- 从「追热点」升级为「造观点」
- 从「PR 执行」升级为「品牌战略指挥」
- 1 人团队 = 20 人全媒体公关效果

### 1.2 架构决策记录

**ADR-001: 技术平台选型**

| 维度 | Dify (v1.0) | OpenClaw (v2.0) |
|------|-------------|-----------------|
| 核心理念 | 工作流自动化 | AI 员工训养 |
| 交互方式 | 可视化节点编排 | 自然语言对话 |
| 灵活性 | 需预先设计流程 | 自主学习适应 |
| 上手难度 | 中等 | 低 |
| 电脑控制 | 需要编程 | 原生支持 |
| 热点监控 | 需要 RSS/API | 自主浏览网页 |

**变更原因**:
1. 单人团队，自然语言交互更高效
2. 老板转发内容多样，需要灵活应对
3. 热点监控需要自主浏览能力
4. 参考傅盛实践，14天可训出可用 Agent 团队

### 1.3 系统边界

```
┌─────────────────────────────────────────────────────────────┐
│                    PineVC-PR 系统                           │
├─────────────────────────────────────────────────────────────┤
│  输入                        │  输出                         │
│  ────                        │  ────                         │
│  • 老板转发文章 (3-5篇/日)   │  • 微信公众号文章              │
│  • 全网热点 (Agent 自主巡逻) │  • 社群早报                    │
│  • 定时任务 (心跳触发)       │  • 投资指引                    │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 Agent 团队设计

#### 大管家 "松禾小助"
```
角色: 团队统筹者
职责:
  • 接收用户指令，分配任务
  • 每 3 小时同步工作汇总
  • 检查各 Agent 工作进度
  • 向用户报告重要事项

技能文件:
  • task_dispatch.md - 任务分发
  • progress_report.md - 进度汇报
  • user_communication.md - 用户沟通
```

#### 笔杆子
```
角色: 内容创作者
职责:
  • 处理老板转发的文章
  • 生成公众号内容
  • 关联核心论点 (CDE+AI)
  • 引用被投企业案例

技能文件:
  • article_process.md - 文章加工
  • thesis_injection.md - 论点注入
  • company_quote.md - 企业引用
  • brand_voice.md - 品牌语调
```

#### 参谋官
```
角色: 情报收集者
职责:
  • 全网巡逻，发现热点
  • 筛选高价值信息
  • 监控行业竞品动态
  • 整理每日信息简报

技能文件:
  • web_patrol.md - 网页巡逻
  • hotspot_filter.md - 热点筛选
  • competitor_monitor.md - 竞品监控
  • source_list.md - 信息源列表
```

#### 运营官
```
角色: 多平台运营者
职责:
  • 适配不同平台格式
  • 管理发布排期
  • 回复评论互动
  • 统计运营数据

技能文件:
  • platform_adapt.md - 平台适配
  • schedule_publish.md - 发布排期
  • engagement_track.md - 互动追踪
```

#### 进化官
```
角色: 质量优化者
职责:
  • 检查其他 Agent 输出质量
  • 发现错误并更新 Skill
  • 学习新的能力
  • 优化工作流程

技能文件:
  • quality_check.md - 质量检查
  • error_learn.md - 错误学习
  • skill_update.md - 技能更新
```

#### 知识官
```
角色: 知识管理者
职责:
  • 维护被投企业信息
  • 更新论点框架
  • 管理品牌规范
  • 整理往期优秀内容

技能文件:
  • portfolio_update.md - 企业更新
  • thesis_update.md - 论点更新
  • knowledge_search.md - 知识检索
```

#### 早报官 "速递官"
```
角色: 社群运营者
职责:
  • 生成每日早报
  • 制作周度盘点
  • 推送到社群
  • 收集社群反馈

技能文件:
  • daily_brief.md - 早报生成
  • weekly_review.md - 周度盘点
  • community_push.md - 社群推送
```

## 3. 核心工作流

### 3.1 WF-B: 老板转发 → 内容加工

```
触发: 用户转发文章链接给大管家
主责 Agent: 笔杆子

流程:
┌─────────────────────────────────────────────────────────────┐
│  1. 大管家接收链接                                           │
│     └→ 记录到 Session，分发给笔杆子                          │
│                                                             │
│  2. 笔杆子读取文章                                           │
│     └→ 使用浏览器访问链接，提取内容                          │
│                                                             │
│  3. 知识官检索相关论点                                       │
│     └→ 匹配 CDE+AI 论点框架                                  │
│                                                             │
│  4. 知识官检索相关企业                                       │
│     └→ 匹配被投企业库                                        │
│                                                             │
│  5. 笔杆子生成内容                                           │
│     └→ 按品牌语调生成公众号文章                              │
│                                                             │
│  6. 进化官质量检查                                           │
│     └→ 检查论点关联、企业引用、品牌规范                      │
│                                                             │
│  7. 大管家汇报给用户                                         │
│     └→ 展示文章，等待确认                                    │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 WF-A: 热点监控 → 内容生成

```
触发: 定时巡逻 (每日凌晨 1:00-6:00)
主责 Agent: 参谋官

流程:
┌─────────────────────────────────────────────────────────────┐
│  1. 参谋官全网巡逻                                           │
│     └→ 访问预设信息源，抓取最新内容                          │
│                                                             │
│  2. 筛选高价值热点                                           │
│     └→ 按相关性、时效性、影响力评分                          │
│                                                             │
│  3. 汇报给大管家                                             │
│     └→ Top 5 热点推荐                                        │
│                                                             │
│  4. 笔杆子生成内容 (如用户确认)                              │
│     └→ 对选中的热点生成 PR 内容                              │
│                                                             │
│  5. 运营官多平台适配                                         │
│     └→ 适配公众号/小红书/头条格式                            │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 WF-E: 每日早报

```
触发: 定时 (每日 8:00)
主责 Agent: 早报官

流程:
┌─────────────────────────────────────────────────────────────┐
│  1. 参谋官整理隔夜新闻                                       │
│     └→ 从巡逻结果中提取                                      │
│                                                             │
│  2. 早报官筛选 Top 5                                         │
│     └→ 按重要性和相关性                                      │
│                                                             │
│  3. 早报官生成早报内容                                       │
│     └→ 按早报模板生成                                        │
│                                                             │
│  4. 用户确认                                                 │
│     └→ 展示预览，等待确认                                    │
│                                                             │
│  5. 推送到社群                                               │
│     └→ 发送到微信群/企微                                     │
└─────────────────────────────────────────────────────────────┘
```

```
  • 每天 3-5 篇文章

  • 更新 Skill 文件
  • 完善工作流程
```

### 5.2 训练心法 (参考傅盛)

```
1. 以 "养新人" 的心态包容试错
   • AI 会犯低级错误
   • 错误 → 规则 → Skill
   • 越试错，越精准

2. 用自然语言交互
   • 不需要代码
   • 像跟员工沟通一样
   • 明确岗位、职责、要求

3. 多互动多沟通
   • 高频交互
   • 让 AI 越来越懂你

4. 聚焦结果，放手探索
   • 只需明确目标
   • 让 AI 自主寻找方法
```

## 8. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| AI 幻觉/错误信息 | 进化官质检 + 人工终审 |
| 保密问题 | 建立保密规则 Skill |
| 定时任务失效 | Heartbeat + Cron 双机制 |
| 记忆混乱 | 三层记忆 + Session 管理 |
| 内容质量不稳定 | 建立质检标准 + 持续迭代 |
