## 第二步：创建工作流

### 创建新工作流

1. 进入「工作室」页面
2. 点击「创建应用」
3. 选择「工作流」
4. 名称: `WF-B: 文章加工`
5. 图标: 选择一个合适的图标

### 配置工作流节点

#### 节点 1: 开始节点 (Start)

点击「开始」节点，添加输入变量：

| 变量名 | 类型 | 标签 | 必填 | 默认值 |
|--------|------|------|------|--------|
| article_url | 文本 | 文章链接 | ✅ | - |
| additional_note | 段落 | 附加说明 | ⬜ | "" |

#### 节点 2: HTTP 请求 (获取文章)

1. 添加「HTTP 请求」节点
2. 连接到「开始」节点
3. 配置:
   - 方法: GET
   - URL: `{{article_url}}`
   - 超时: 30 秒
   - 输出变量: **自动生成**（包括 `body`、`status_code`、`headers` 等）

> **注意**:
> - HTTP 请求节点的输出变量名是**系统固定**的
> - 后续节点通过 `{{HTTP请求.body}}` 引用网页正文
> - 微信公众号文章可能需要代理获取，可以使用 n8n 作为代理

#### 节点 3: 知识库检索 (论点)

1. 添加「知识库检索」节点
2. 连接到 HTTP 请求节点
3. 配置:
   - 知识库: `thesis-framework`
   - 查询内容: `{{HTTP请求.body}}`
   - Top K: 3
   - 输出变量: **自动生成，无需手动设置**（默认为 `知识库检索.result`）

> **注意**: 知识库检索节点的输出变量名是**系统固定**的，无法修改。后续节点通过 `{{知识库检索.result}}` 引用。

#### 节点 4: 知识库检索 (企业)

1. 添加「知识库检索」节点
2. 连接到上一个知识库节点
3. 配置:
   - 知识库: `portfolio`
   - 查询内容: `{{HTTP请求.body}} {{知识库检索.result}}`
   - Top K: 3
   - 输出变量: **自动生成**（默认为 `知识库检索_1.result`，具体名称看节点列表）

#### 节点 5: LLM (内容生成)

1. 添加「LLM」节点
2. 连接到知识库节点
3. 配置:
   - 模型: 智谱 AI / GLM-4-Flash
   - 系统提示词: (复制 `src/prompts/article-generation-v2.md` 中的系统提示词)
   - 用户提示词:
```
请根据以下信息生成一篇松禾医健的公众号文章。

## 原始文章信息
**链接**: {{article_url}}
**正文**:
{{HTTP请求.body}}

## 附加说明
{{additional_note}}

## 检索到的论点参考
{{知识库检索.result}}

## 可引用的被投企业
{{知识库检索_1.result}}

请按照标准格式输出文章，包含：标题、核心要点、事件背景、行业影响、松禾视角、我们的布局、标签。
```

> **变量说明**: Dify 中引用变量格式为 `{{节点名称.变量名}}`，实际节点名称以你工作流中显示的为准。
   - 输出变量: `generated_article`

#### 节点 6: LLM (质量检查)

1. 添加「LLM」节点
2. 连接到内容生成节点
3. 配置:
   - 模型: 智谱 AI / GLM-4-Flash
   - 系统提示词: `你是内容质量审核员。请检查文章是否符合品牌规范。`
   - 用户提示词:
```
请检查以下文章是否符合松禾医健的品牌规范:

{{generated_article}}

检查项:
1. 篇幅是否在 600-2000 字
2. 是否包含禁用词 (完爆、碾压、秒杀等)
3. 是否包含论点关键词
4. 结构是否完整

输出 JSON 格式的审核结果。
```
   - 输出变量: `quality_report`

#### 节点 7: 结束节点

1. 连接质量检查节点到「结束」节点
2. 配置输出变量:
   - `title`: 从生成文章中提取标题
   - `content`: `{{generated_article}}`
   - `quality`: `{{quality_report}}`

## 第三步：测试工作流

### 运行测试

1. 点击右上角「运行」按钮
2. 输入测试链接:
   - 可以使用一篇医药行业新闻
   - 例如: `https://example.com/pharma-news`
3. 点击「开始运行」
4. 观察各节点执行情况
5. 检查输出结果

### 调试技巧

- 如果知识库检索结果不理想，检查分段设置
- 如果 LLM 生成内容不符合预期，调整提示词
- 如果 HTTP 请求失败，检查 URL 是否可访问

## 第四步：发布 API

### 获取 API 端点

1. 点击「发布」按钮
2. 选择「API」方式
3. 复制 API 端点和密钥

### API 调用示例

```python
import requests

url = "http://localhost:5001/v1/workflows/run"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
data = {
    "inputs": {
        "article_url": "https://example.com/article",
        "additional_note": "重点突出临床效率"
    },
    "user": "test-user"
}

response = requests.post(url, headers=headers, json=data)
print(response.json())
```

## 常见问题

### Q: 微信文章无法获取？
A: 微信公众号有防盗链机制，需要使用 n8n 代理或手动粘贴内容。

### Q: 知识库检索结果不准确？
A: 检查分段设置，尝试调整 Top K 值，或优化查询语句。

### Q: LLM 生成内容太长/太短？
A: 在提示词中明确篇幅要求，如"正文 800-1500 字"。

### Q: 如何更新知识库？
A: 在知识库页面上传新文档或编辑现有文档，系统会自动重新索引。
