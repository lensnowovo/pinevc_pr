# Skill: 质量检查 (quality_check)

> 进化官核心技能

## 触发条件

- 其他 Agent 完成内容生成后
- 用户要求检查内容时

## 检查清单

### 1. 论点关联检查

```
检查项:
□ 是否关联了至少 1 个核心论点?
□ 论点关联是否自然，不生硬?
□ 论点引用的叙事是否准确?

评分: 1-5 分
不通过标准: < 3 分
```

### 2. 企业引用检查

```
检查项:
□ 引用的企业信息是否准确?
□ 企业名称是否正确? (全称/简称)
□ 企业进展是否最新?
□ 引用是否自然，不突兀?

评分: 1-5 分
不通过标准: < 3 分
```

### 3. 品牌规范检查

```
检查项:
□ 语调是否符合专业、前瞻的品牌形象?
□ 是否使用了禁用词? (参考 blacklist.md)
□ 标题是否夸张/标题党?
□ 是否有错别字/语病?

评分: 1-5 分
不通过标准: < 3 分
```

### 4. 内容质量检查

```
检查项:
□ 逻辑是否通顺?
□ 数据是否有来源?
□ 观点是否有依据?
□ 字数是否在 800-1500 范围?

评分: 1-5 分
不通过标准: < 3 分
```

### 5. 合规检查

```
检查项:
□ 是否涉及未公开信息?
□ 是否有股价/市值预测?
□ 是否有竞品不当对比?
□ 是否有医疗效果承诺?

一票否决: 任一项不通过
```

## 检查报告模板

```markdown
# 📋 质量检查报告

**检查对象**: {文章标题}
**检查时间**: {时间}
**检查者**: 进化官

---

## 检查结果

| 检查项 | 得分 | 状态 |
|--------|------|------|
| 论点关联 | 4/5 | ✅ |
| 企业引用 | 3/5 | ⚠️ |
| 品牌规范 | 5/5 | ✅ |
| 内容质量 | 4/5 | ✅ |
| 合规检查 | - | ✅ |

**综合评分**: 4.0/5.0
**最终结论**: ✅ 通过 / ❌ 不通过

---

## 问题详情

### ⚠️ 企业引用 (3/5)
- 问题: 引用的 XX 公司进展不是最新
- 建议: 更新为 2026 年 2 月的最新动态

---

## 修改建议

1. {具体修改建议 1}
2. {具体修改建议 2}

---

## 是否可以发布?

- [ ] ✅ 可以发布
- [ ] ⚠️ 修改后可发布
- [ ] ❌ 需要重写
```

## 处理流程

```
检查结果
    │
    ├── ✅ 全部通过 → 标记为"可发布"，通知大管家
    │
    ├── ⚠️ 小问题 → 记录问题，通知原作者修改
    │
    └── ❌ 严重问题 → 退回重写，记录到错误学习库
```

## 错误学习

当发现新类型的错误时：

```
1. 分析错误原因
2. 提炼为规则
3. 更新相关 Skill 文件
4. 通知所有相关 Agent

格式:

## 错误学习记录

| 日期 | 错误类型 | 错误描述 | 新增规则 |
|------|----------|----------|----------|
| 2026-03-01 | 企业引用 | 引用了已退出的企业 | 检查前先验证企业状态 |
```

## 注意事项

1. **严格但合理**: 不是吹毛求疵，而是确保质量
2. **建设性反馈**: 给出具体修改建议，不只是指出问题
3. **持续学习**: 每次检查都是优化规则的机会
